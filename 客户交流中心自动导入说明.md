# QUAD 客户交流中心专用导入接口

此接口供负责 Meta 与 Yelp 客资提取的两台电脑使用。新客资先进入“客户交流中心”，只有店内人员把跟进状态改成“已邀约”或“已预约”后，才进入“高意向客户”。

## 接口

- 地址：`POST https://film-shop-management-production.up.railway.app/api/import/customer-conversations`
- 鉴权：请求头 `X-Import-Token`
- 禁止使用旧地址 `/api/import/prospects`，旧地址会直接写入高意向客户。
- 每台电脑使用独立密钥；密钥只允许导入客资，不能读取财务或操作其他模块。

请求示例（字段可以按实际情况增减）：

```json
{
  "importSource": "codex",
  "sourceDevice": "meta-mac",
  "items": [{
    "externalId": "meta-thread-123",
    "source": "Meta / Facebook",
    "customer": "Customer Name",
    "phone": "+17025550123",
    "vehicle": "2025 Tesla Model Y",
    "need": "全车窗膜，询问价格",
    "profileUrl": "https://...",
    "chatContext": "客户留下电话并询问全车窗膜价格",
    "conversationMessages": [
      { "speaker": "customer", "text": "How much for tint?", "timestamp": "2026-07-13T21:00:00-07:00" },
      { "speaker": "staff", "text": "What vehicle do you have?", "timestamp": "2026-07-13T21:01:00-07:00" }
    ]
  }]
}
```

`externalId` 应使用平台稳定的 lead/thread/conversation ID。同一个 `externalId` 再次上传会更新原客户并合并新聊天，不会重复建客户。没有平台 ID 时，系统会用来源和电话号码等信息去重。

## Windows 两台电脑（店主在每台电脑各操作一次）

Meta Windows 电脑使用 Railway 中的 `CUSTOMER_CONVERSATION_IMPORT_TOKEN_META`；Yelp Windows 电脑使用 `CUSTOMER_CONVERSATION_IMPORT_TOKEN_YELP`。两把密钥不能混用。

在每台电脑的项目目录打开 PowerShell，由店主复制该电脑对应的专用密钥，然后运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\tools\windows-store-quad-import-token.ps1
```

脚本会让店主在隐藏输入框中粘贴密钥，并保存到当前 Windows 用户的 DPAPI 加密文件：

```text
%APPDATA%\QUAD\customer-import-token.dat
```

这个文件不是明文，复制到另一台电脑或换成另一个 Windows 用户也不能解密。不要把密钥直接写入 PowerShell 命令、环境变量、代码、JSON、聊天或 Codex 记忆。

## 自动上传程序

客资提取程序把待传数据原子写入 `imports/customer-conversation-outbox.json`（先写临时文件，再重命名）。然后运行：

```powershell
$env:SOURCE_DEVICE = "meta-windows"
node .\tools\customer-conversation-import-loop.mjs
```

Yelp Windows 电脑运行：

```powershell
$env:SOURCE_DEVICE = "yelp-windows"
node .\tools\customer-conversation-import-loop.mjs
```

程序会自动读取本机 DPAPI 加密密钥，每 30 秒检查一次。成功后才把 outbox 移入 `imports/processed/`；网络或服务器失败会保留文件并自动重试，因此之前暂存的客资也可以批量补传。

## 给两台电脑 Codex 的任务要求

持续监控各自平台。当客户留下电话或其他有效联系方式时，立即把客户资料、车辆/需求、平台链接、平台稳定 ID 和已有完整聊天写入 outbox。使用上述专用程序上传到客户交流中心。不得调用旧的高意向客户接口，不得把密钥读取后显示、记录或写入任何文件。
