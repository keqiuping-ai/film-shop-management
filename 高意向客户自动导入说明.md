# 高意向客户自动导入说明

这个方案用于 Mat、Yelp、Meta、Google 等平台的客资自动进入系统。每台电脑可以各自抓一个平台，统一写入云端的“高意向客户”模块。

## 导入接口

- 地址：`POST /api/import/prospects`
- 权限：推荐使用专用导入密钥。服务器支持旧的 `PROSPECT_IMPORT_TOKEN`，也支持新的多密钥 `PROSPECT_IMPORT_TOKENS`。也可以使用有“录入/编辑高意向客户”权限的登录账号。
- 建议频率：每 6 分钟导入一次。
- 单次上限：200 条。

## 安全密钥

服务器可以配置一个旧密钥：

```bash
PROSPECT_IMPORT_TOKEN="一串足够长的随机密钥"
```

也可以给每台平台电脑配置独立密钥：

```bash
PROSPECT_IMPORT_TOKEN_YELP="Yelp电脑密钥"
PROSPECT_IMPORT_TOKEN_META="Meta电脑密钥"
PROSPECT_IMPORT_TOKEN_MAT="Mat电脑密钥"
```

也可以配置多个电脑专用密钥，多个密钥用换行、逗号或分号分开：

```bash
PROSPECT_IMPORT_TOKENS="yelp电脑密钥
meta电脑密钥
mat电脑密钥"
```

抓客资电脑导入时，在请求头里带上：

```http
X-Import-Token: 一串足够长的随机密钥
```

这样每台抓取电脑不需要保存老板账号密码。这个密钥只允许导入“高意向客户”，不能读取或修改其它系统模块。

建议每个平台电脑单独一个密钥。比如 Meta 电脑只用 Meta 密钥，Yelp 电脑只用 Yelp 密钥。以后某台电脑停用时，只需要删除那台电脑的密钥，不影响其它电脑。

## 去重规则

系统会按下面顺序判断是否重复：

1. `externalId` / `conversationId` / `profileUrl` 加来源平台。
2. 客户电话。
3. 来源平台 + 客户姓名 + 车型或需求。
4. 来源平台 + 客户姓名。

重复时不会新建第二条，会把新的聊天上下文和备注合并到原客户里面。

```json
{
  "importSource": "codex",
  "sourceDevice": "front-desk-mac-meta",
  "items": [
    {
      "source": "Meta / Facebook",

      "externalId": "meta-conversation-123"
    }

}
```

## 每台电脑的导入脚本

把 Codex 抓到的客资写入 `imports/prospect-outbox.json`，然后运行：

```bash
cd "/Users/keqiuping/Desktop/美国贴膜店管理系统/联网版-app"
PROSPECT_IMPORT_TOKEN="本电脑导入密钥" \
SOURCE_DEVICE="meta-office-1" \
PROSPECT_OUTBOX_FILE="imports/prospect-outbox.json" \
node tools/prospect-import-loop.mjs
```

脚本会每 6 分钟检查一次文件。导入成功后，会把文件移到 `imports/processed/`，避免重复提交。

如果只想测试一次：

```bash
PROSPECT_IMPORT_TOKEN="导入密钥" node tools/prospect-import-loop.mjs --once
```

## Meta 电脑固定字段

Meta 电脑建议使用：

```text
sourceDevice = meta-office-1
importSource = codex
source = Meta / Facebook
```

导入时不要用鼠标键盘填系统表格，抓到客资后直接调用 `POST /api/import/prospects`。
