# 系统升级通道

这个目录预留给后续系统升级使用。

当前第一版只做安全的升级框架，不自动执行远程代码。

已经预留：

- `version.json`：当前系统版本。
- `server-config.json` 里的 `update` 配置：升级通道、升级检查地址、是否允许远程升级。
- `/api/system/info`：查看当前系统版本和升级配置。
- `/api/system/check-update`：后续连接云端升级服务时使用。
- `migrations/`：后续数据库结构升级脚本目录。

正式接入远程升级时必须加入：

1. 管理员权限确认。
2. HTTPS 升级源。
3. 更新包签名校验。
4. 升级前自动备份。
5. 升级失败自动回滚。
6. 升级日志。

不要直接下载并执行未知脚本。
